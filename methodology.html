Setup Quality Measurement Protocol v3.0

How Setup Quality
Is Measured.

Four rules. Binary pass/fail. Computed per trade. Auditable against any closed position.

Setup Quality is not a subjective judgment. It is a binary score against four non-negotiable risk rules, computed per trade from actual trade data.

All four rules must pass for a trade to count as setup-valid. Any single failure makes it setup-invalid — regardless of whether the trade made money.

A profitable trade that violated the rules still counts as setup-invalid. A losing trade that followed all four rules still counts as setup-valid.

This is intentional. The point is the process, not the P&L.

The Four Rules

R1

Max Loss Per Trade

2% – 3% of account equity

Every trade sized so the maximum loss (stop distance x position size) is between 2% and 3% of current account equity, inclusive. Below 2% is undersized. Above 3% is unacceptable blowup risk. Default stop is 10 points.

risk_pct = abs(entry_price - stop_price) x contracts x point_value / account_equity

PASS if 0.02 ≤ risk_pct ≤ 0.03
FAIL otherwise

Edge cases: Stop moved to breakeven mid-trade → original risk at entry counts. Stop widened during trade → original stop at entry counts. No stop data → default 10-point stop used for calculation.

R2

Minimum Profit on Winners

≥1% of account equity

Every closed winning trade must realize a profit of at least 1% of account equity. Losers and breakeven trades are exempt — Rule 1 handles the downside. Without this rule, a trader takes premature scalps while letting losses grow.

realized_profit_pct = trade_pnl / account_equity

For winners (pnl > 0):
  PASS if realized_profit_pct ≥ 0.01
  FAIL if realized_profit_pct < 0.01

For losers/breakeven (pnl ≤ 0):
  N/A — auto-pass (rule does not apply)

Note: This rule is outcome-based, not intent-based. It measures whether winners were held long enough to be meaningful, using data already captured at trade close.

R3

Minimum Cooldown After Loss

60 minutes

After any closed losing trade, no new trade may be entered for 60 minutes. Revenge trading is the #1 cause of blowups. A 60-minute cooldown breaks the emotional reactivity loop.

If previous trade P&L < 0:
  elapsed = current_entry_time - previous_close_time
  PASS if elapsed ≥ 60 minutes
  FAIL if elapsed < 60 minutes

If previous trade P&L ≥ 0:
  PASS automatically

Edge cases: Multiple losses in sequence → cooldown resets from most recent loss. Breakeven trade → not a loss, no cooldown required.

R4

Aggregate Limits Intact at Entry

Daily <5%, Weekly <10%

At entry, cumulative daily losses have not hit the 5% daily cap, and cumulative weekly losses have not hit the 10% weekly cap. Caps are meaningless if violated.

daily_loss_pct = sum(today's losses) / account_equity
weekly_loss_pct = sum(this week's losses) / account_equity

PASS if daily_loss_pct < 5% AND weekly_loss_pct < 10%
FAIL if either cap breached at entry

The Scoring Formula

For every trade:
  setup_valid = R1_pass AND R2_pass AND R3_pass AND R4_pass
  (All four must pass. No partial credit.)

Setup Quality % = (forward trades with setup_valid) / (total forward trades) x 100

Tracking start date: April 18, 2026
Trades before this date: annotated per-rule but NOT counted in aggregate %

"No partial credit. Three of four does not count. Adherence is absolute."

Setup Quality is computed on all trades entered on or after April 18, 2026 — the date the measurement protocol was formally adopted. Earlier trades are still annotated per-rule in the trade log but do not affect the scorecard percentage.

Account Equity Method

Account equity at each trade entry is approximated using:

equity_at_entry = $10,000 (starting balance) + cumulative_P&L_before_this_trade

This is sufficient for a small proof-of-concept account. Automated Tradovate API snapshots may replace this in a future phase.

How This Is Verified

1. Data Source

All trade data comes from fills posted publicly in the Ekantik Discord — entry price, stop price, direction, and realized P&L. Account equity is tracked as starting balance plus cumulative P&L.

2. Rule Computation

Each rule is computed by a deterministic formula (published above) applied to the trade's data. There is no subjective scoring step. The same inputs always produce the same output.

3. Public Audit Trail

Every trade in the live trade log shows its per-rule breakdown. Any visitor can verify any Setup Quality claim by checking the trade log against the rules on this page.

If a trade in the log is marked setup-valid but the data shows a rule violation, that is a material public error that will be corrected with timestamped disclosure.

Version History

v3.0 — April 2026

Initial protocol. Four rules as above. Outcome-based Rule 2. All trades scoreable from inception.

No changes to rules may be made during the 100-trade commitment without public disclosure and a version bump.

The Commitment

These four rules were written once. They will not change during the 100-trade consistency proof. If I find a better rule, a better threshold, or a better measurement approach, I will note it publicly — but I will not apply it retroactively. The 100 trades are committed to the rules as written above.

This is not because the rules are perfect. It is because mid-commitment rule changes destroy the commitment.

After 100 trades, the rules can evolve. Until then, they are frozen.

— The Operator
